Làm task $ARGUMENTS theo tasks.md của feature tương ứng.
Trình kế hoạch trước khi sửa bất kỳ file nào: contract cần đổi gì (schema/topic — theo skill
protocol-guard), test nào viết trước (naming theo skill test-evidence), file nào sẽ chạm.
Chờ xác nhận rồi mới thực hiện. Kết thúc bằng make verify.
